## 1. VIDEO CONTENT REQUIREMENTS
## =============================

✅ INTRODUCTION
   Requirement: The video must start with a unique, AI-generated image 
                that captures the essence of the uploaded media.
   Implementation: ImageGenerator.generateIntroImage()
   - Generates image using Gemini API based on media summary
   - Prompt: Creates cinematic portrait travel poster
   - Located in VideoGenerationWorker (Step 2, ~12% progress)
   Status: VERIFIED

✅ MAIN CONTENT - CHRONOLOGICAL ORDER
   Requirement: Photos and videos must be displayed from oldest to newest.
   Implementation: MediaProcessor.scanAndSort()
   - Scans folder for supported formats
   - Sorts by modification date in ascending order
   - Uses Files.list() + sorting
   Status: VERIFIED

✅ MAIN CONTENT - ORIENTATION
   Requirement: All media must be shown in the correct orientation.
   Implementation: ImageProcessor.processImage()
   - Reads EXIF orientation data
   - Applies all 8 rotation cases (0°, 90°, 180°, 270°, flip, etc.)
   - Handles missing EXIF data gracefully
   Status: VERIFIED

✅ MAIN CONTENT - SIZE ADJUSTMENT
   Requirement: All media must be size-adjusted to fill the screen while 
                maintaining their original aspect ratios.
   Implementation: 
   - ImageProcessor: Uses Thumbnails.keepAspectRatio(true) + cropToFill
   - VideoProcessor: Extracts frames at target resolution with aspect ratio
   - Both scale to 1080x1920 (portrait)
   Status: VERIFIED

✅ MAIN CONTENT - AUDIO DESCRIPTIONS
   Requirement: Each photo and video must have generated audio/voice 
                describing it.
   Implementation: 
   - TextGenerator.describeMedia() generates 2-sentence description via Gemini
   - TTSGenerator converts text to speech using Text-to-Speech engine
   - AudioNormalizer applies YouTube audio standards
   - Each media file gets unique audio file in temp directory
   Status: VERIFIED

✅ CONCLUSION - MAP WITH GPS LOCATIONS
   Requirement: Final image must be a map pinpointing GPS locations of 
                the very first and very last photo or video.
   Implementation: MapRenderer.renderMap()
   - Generates AI map via ImageGenerator.generateMapImage()
   - Uses first media's GPS for start location
   - Uses last media's GPS for end location
   - Prompt includes visual markers and route line
   Status: VERIFIED

✅ CONCLUSION - DISTINCT PINPOINTS
   Requirement: The two pinpoints must be different and recognizable.
   Implementation: 
   - Map generation prompt specifies:
     • Green marker at starting location
     • Red marker at ending location
     • Dotted/dashed line connecting points
     • Clear labels (START/END)
   - Different colors + distinct styling ensures visibility
   Status: VERIFIED

✅ CONCLUSION - AI-GENERATED TEXT & INSPIRATIONAL PHRASE
   Requirement: Map image must include AI-generated text featuring an 
                inspirational phrase based on the specific locations visited.
   Implementation: 
   - TextGenerator.generateInspirationalPhrase() generates phrase via Gemini
   - Phrase is based on actual place names (geocoded from GPS)
   - MapRenderer.overlayPhrase() renders text on map with:
     • Font: Bold sans-serif 52pt
     • Color: White with shadow
     • Background: Semi-transparent black
   - GeocodingService converts GPS → place names via Nominatim API
   Status: VERIFIED

## 2. VIDEO FORMAT REQUIREMENTS
## =============================

✅ PORTRAIT MODE (1080x1920)
   Requirement: Resulting video must be in portrait orientation.
   Implementation: 
   - config.properties: output.resolution=1080x1920
   - VideoAssembler: Sets frame size to 1080x1920
   - All processors scale to 1080x1920
   - Output MP4: H.264 video, AAC audio, 30 fps
   Status: VERIFIED

✅ AUDIO STANDARDS - LOUDNESS
   Requirement: Loudness under YouTube standard (-16 to -14 LUFS)
   Implementation: AudioNormalizer targets -15 LUFS
   - Uses FFmpeg's loudnorm filter
   - Two-pass normalization:
     1. First pass: measure loudness
     2. Second pass: apply gain correction
   - Measured loudness value within YouTube spec
   Status: VERIFIED

✅ AUDIO STANDARDS - TRUE PEAK
   Requirement: True peak under mastering (-2 to -1 dBTP)
   Implementation: AudioNormalizer targets -1.5 dBTP
   - FFmpeg's loudnorm includes peak limiting
   - Measured peak value within spec
   - Prevents clipping during playback
   Status: VERIFIED

✅ AUDIO STANDARDS - LRA (LOUDNESS RANGE)
   Requirement: LRA under YouTube standard (5-10 LU)
   Implementation: AudioNormalizer measures and validates LRA
   - FFmpeg loudnorm outputs measured LRA
   - Falls within YouTube spec range
   - Ensures consistent dynamics across video
   Status: VERIFIED
